    1. Общие сведения.

    ПDE является заменителем оболочки для операционных системм OS/2 и eCS. Она
разработана так, что бы не конфликтовать с WPS, включая любой WPS-енхансер, 
такой как XWorkPlace, Nice, CandyBarZ, eStyler/2 и т.д..
    Я не имел проблемм с CandyBarZ, и с Nice.
    ПDE может использоватся как целиком, так и покомпонентно.
    ПDE называется так, потому-что это Desktop Environment (Десктоп Оболочка).
Плюс "Полумух" -> OS/2 :-)


    2. Установка.

    После установки ПDE предоставляет вам другой Десктоп, пусковую Панель и
Файл-менеджер ("ПолуФайл"). ПDE не зависит от  целосности OS2.INI и OS2SYS.INI.
    Простейший случай установить ПDE на данный момент это:
  + распаковать архив в C:\PDE (*)
  + в вашем CONFIG.SYS изменить строку с 
    SET RUNWORKPLACE= на файл PDM.EXE
    (вы найдете его в C:\PDE\), таким образом:
    SET RUNWORKPLACE=C:\PDE\PDM.EXE
  + перезагрузится

(*) Вы можете поместить ПDE в любую другую папку, но тогда нужно добавить в
CONFIG.SYS строку SET PDE_HOME=[ваша_папка]
    После перезагрузки отконфигурируйте ПDE на свой вкус.

    3. Конфигурация.

    Все установки ПDE сохраняются в папке [PDE_HOME]\PDEConf\ и являются
простыми текстовыми файлами с расширением "cfg", где:

//это строка коментария

и

Ключ=Значение

так что вы можете просто их редактировать.

   Сейчас я пишу графическую (GUI) программу конфигурации ПDE и скоро она
увидит свет.
Вот описание для .CFG файлов:
  general.cfg  - настройки, общие для всех программ ПDE, язык, например.
  pdm.cfg      - установки Десктопа.
  filehalf.cfg - установки ПолуФайла, файлового менеджера ПDE.
  deskhalf.cfg - стартовая панель (я называю  её П-Панелью).

  desktop.progs и ppanel.progs - здесь указываются элементы на Десктопе и П-Панели.

  quickrun.cfg - программы для меню "Быстрый запуск" ПолуФайла.

  openwith.cfg - программы для пункта меню "Открыть с" ПолуФайла.

  associations.cfg - Я думаю, основной. Здесь указаны типы файлов для ПолуФайла.

    4. Известные проблеммы.

Во-первых, мой Русский (и Английский). Меня часто тыкают носом в ошибки, _но_
я не Русский и не Англичанин, что-бы хорошо знать эти языки.
Поверьте, Украинский язык я знаю очень хорошо.
Во-вторых -- нежелание юзеров сначала почитать доки и лишь потом запускать
программу и спрашивать "а по-чему не работает?"
:-)
Иногда ПDE не устанавливает рабочую директорию  для програм  и это действительно
проблемма для старых програм (например DoomII, Quake, :-( ).
П-Панель иногда замерзает, когда не неё переключаются из Списка Окон, так что
будьте готовы её прибить Котом.
ПолуФайл может улететь при просмотре свойств файлов и папок с диска CD-ROM.
